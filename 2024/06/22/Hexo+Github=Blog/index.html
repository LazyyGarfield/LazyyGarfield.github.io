Blog的制作过程

介绍如何形成个人的Blog

第一步

安装Nodejs和Git

  1. 安装Nodejs，下载10.15.1LTS并安装，其中一直点击next即可
  2. 安装git，安装git的详细步骤百度即可

第二步

安装hexo框架

  1. 打开终端，win10中的操作为以管理员的身份打开【win+R+cmd】，以下均为在终端中输入的命令
  2. 输入命令：【node -v】和【npm -v】查看node和npm版本，测试其是否安装成功
  3. 安装淘宝的cnpm管理器，命令为【npm install -g cnpm –registry=http://registry.npm.taobao.org】
  4. 输入命令【cnpm -v】查看cnpm版本
  5. 输入【cnpm install -g hexo-cli】 安装hexo框架
  6. 用命令【hexo -v】查看hexo版本

第三步

搭建hexo框架

  1. 在终端【cmd】中，进入到自己选择的文件夹目录
  2. 输入命令【mkdir blog】创建blog目录
  3. 输入【cd blog】进入到blog中，我们以下的所有操作均在blog文件夹中进行
  4. 输入【hexo init】搭载hexo框架，如果卡在【INFO Install dependencies】可以输入【Ctrl+C】退出，然后输入【cnpm install】
  5. 输入【hexo s】生成本地的初始化博客，可以在浏览器中输入地址【localhost:4000】查看，然后回到终端【cmd】中，输入【Ctrl+C】和【Y】退出该步
  6. 输入【hexo n “我的第一篇博客”】创建新的文章，系统会在【blog/source/posts】下创建一个.md文件，可以在.md文件中进行编辑，语法为markdown语法
  7. 依次输入【hexo clean】和【hexo g】更新博客
  8. 重复步骤6，登入本地博客查看

第四步（最重要的一步）

将hexo部署到github中，形成远端博客

  1. 登录到github中创建一个新的仓库【XXX.github.io】，其中【XXX】必须为github的用户名

  2. 在终端【cmd】中输入命令【cnpm install –save hexo-deployer-git】，安装git部署插件，其中忽略warning

  3. 在【/blog】中找到文件【_config.yml】，在其中找到【# Deployment】并修改为如下代码

    # Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
type: git
repo: https://github.com/XXX/XXX.github.io.git
branch: master

    其中【XXX】仍然是github的用户名

  4. 回到终端【cmd】，输入命令【git config –global user.email “YYY”】和【git config –global user.name “XXX”】，这里的【YYY】和【XXX】分别填写github账号的邮箱和用户名

  5. 在终端上输入【hexo d】，就可以将博客部署到远端啦

  6. 在浏览器上登陆【https://github.com/XXX/XXX.github.io】进入到github仓库，其中【XXX】依然是github的用户名，可以发现在仓库上已经生成了一些东西

  7. 最终，在浏览器上登陆【https://XXX.github.io】，初次进入尝试多次刷新，等待博客部署到远端上，这个就是博客地址啦！

更新博客

  1. 在每次更新博客时，在终端上先cd到【blog】目录中，输入【hexo n “XXX”】，其中【XXX】为博客的名字，系统则会在【blog/sources/_posts】下创建一个【XXX.md】文件，用户可以在md文件中直接编辑，语法为markdown语法
  2. 在终端上依次输入【hexo clean】和【hexo g】
  3. 这一步可以选做，在终端上输入【hexo s】则可以直接在本地【https://localhost:4000】观察效果
  4. 可以跳过第3步，直接在终端输入【hexo d】更新博客
  5. 使用【hexo d】更新博客时，可能会卡住或出现【Spawn failed】错误，该错误的原因可能为访问github.com的网络域名问题，等待一段时间后再尝试或者多尝试几次即可
