MyTinySTL底层

阅读顺序

从第一层到第八层依次为：

仿函数->迭代器->算法->分配器（模板）->适配器->容器

底层（头文件）

底层中的元素不是项目里的文件，而是文件中用到的各个头文件。建议在阅读源码时遇到头文件再去查它的作用

cstddef

定义了常用的常量、宏、类型和函数

stdexcept

包含stdexcept这个头文件，定义了一些标准异常类，这些异常类通常被用于表示 C++ 程序在运行时的错误

cassert

定义了一个 assert 函数，可以用于异常判断

const noexcept

表示函数是一个常量成员函数并且在运行时不会引发异常。这对于哈希函数是一种良好的属性，因为哈希函数应该是高效和可预测的。

new

new的头文件中只包含6个重载的new和delete，和一些new的声明

pragma warning

  1. #pragma warning只对当前文件有效（对于.h，对包含它的cpp也是有效的），而不是对整个工程的所有文件有效。当该文件编译结束，设置也就失去作用。
  2. #pragma warning(push)是保存当前的编译器警告状态；#pragma warning(push, n) 存储当前报警设置，并设置报警级别为n。n为从1到4的自然数。
  3. #pragma warning(pop) 恢复之前压入堆栈的报警设置。该句之后的代码，就恢复了warning，即在一对push和pop之间作的任何报警相关设置都将失效。
  4. #pragma warning(disable: n) 将某个警报置为失效。
  5. #pragma warning(default: n) 将报警置为默认。
  6. #pragma warning (enable: n)将被禁用的warning n启用
  7. #pragma warning (error: n)将一个warning作为一个错误
  8. once:只显示一次(警告/错误等)消息
